  <behavior>
    - SetSplitRatio: Given a tree with a SplitNode, setting ratio by NodeId returns tree with updated ratio
    - SetSplitRatio: Ratio is clamped to MinRatio (0.1) and MaxRatio (0.9)
    - SetSplitRatio: Non-existent NodeId returns unchanged tree (ReferenceEquals)
    - SetSplitRatio: Works on nested split nodes (finds correct node in deep tree)
    - SwapPanes: Given tree with two leaves A and B, swapping produces tree where A's position has B's data and vice versa
    - SwapPanes: Tree structure (SplitNodes, axes, ratios) is preserved -- only leaf content changes
    - SwapPanes: Swapping a pane with itself returns unchanged tree
    - SwapPanes: Non-existent paneId returns unchanged tree
    - MovePaneToTarget: Source leaf is removed and re-inserted adjacent to target with correct axis
    - MovePaneToTarget: Direction.Left/Right creates Vertical split; Direction.Up/Down creates Horizontal split
    - MovePaneToTarget: Direction.Left/Up inserts source as First child; Direction.Right/Down inserts source as Second child
    - MovePaneToTarget: Source == target returns unchanged tree
    - MovePaneToTarget: When source is the only pane, returns unchanged tree (cannot remove last pane)
  </behavior>
  <action>
    RED phase: Create tests/Wcmux.Tests/Layout/PaneInteractionTests.cs with xUnit tests covering all behaviors above. Build test trees using LeafNode and SplitNode records directly. Run tests -- all must FAIL (methods don't exist yet).

    GREEN phase: Add to LayoutReducer.cs:

    1. `SetSplitRatio(LayoutNode root, string nodeId, double newRatio)` -- recursive tree walk finding SplitNode by NodeId, returns `split with { Ratio = Math.Clamp(newRatio, MinRatio, MaxRatio) }`. Uses same ReferenceEquals short-circuit pattern as existing methods.

    2. `SwapPanes(LayoutNode root, string paneIdA, string paneIdB)` -- finds both LeafNodes first, then produces new tree swapping their PaneId, SessionId, Kind. Implementation: first do a tree walk to find leafA and leafB data, then do a second tree walk replacing leafA data with leafB data and vice versa. If either pane not found or same pane, return root unchanged.

    3. `MovePaneToTarget(LayoutNode root, string sourcePaneId, string targetPaneId, Direction dropSide)` -- Composition of ClosePane + SplitPane. Steps: (a) find source leaf data, (b) call ClosePane to remove source, (c) if result is null return root (was only pane), (d) determine axis from direction (Left/Right = Vertical, Up/Down = Horizontal), (e) determine insert order (Left/Up = source is First, Right/Down = source is Second), (f) insert source adjacent to target using a private InsertPaneAtTarget helper. If source == target, return root unchanged.

    The InsertPaneAtTarget helper walks the tree to find the target LeafNode and wraps it in a new SplitNode with the source as sibling (similar to SplitPaneInTree but takes an existing LeafNode instead of creating one).

    Also add private helper `FindLeaf(LayoutNode root, string paneId)` returning `LeafNode?` for use by SwapPanes and MovePaneToTarget.

    Run tests -- all must PASS.

    REFACTOR: Ensure FindLeaf is reused where possible. No duplication with existing private helpers.
