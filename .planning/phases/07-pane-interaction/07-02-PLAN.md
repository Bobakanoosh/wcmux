  <files>src/Wcmux.App/Views/CursorBorder.cs, src/Wcmux.App/Views/WorkspaceView.xaml.cs, src/Wcmux.App/Commands/PaneCommandBindings.cs</files>
  <action>
    **CursorBorder.cs** -- Create new file in src/Wcmux.App/Views/:
    ```
    namespace Wcmux.App.Views;
    internal sealed class CursorBorder : Border
    {
        public InputCursor Cursor { get => ProtectedCursor; set => ProtectedCursor = value; }
    }
    ```
    Uses `Microsoft.UI.Input.InputCursor` and `Microsoft.UI.Xaml.Controls.Border`.

    **WorkspaceView.xaml.cs** -- Add mouse resize handles:

    1. Add field: `private readonly List<UIElement> _resizeHandles = new();`

    2. Add private method `CreateResizeHandles()` that:
       - Clears existing handles from RootContainer.Children and the list
       - Walks the layout tree to find all SplitNodes
       - For each SplitNode, computes the boundary position from PaneRects of its children
       - Creates a CursorBorder (6px wide for vertical splits, 6px tall for horizontal splits)
       - Sets transparent background (Color.FromArgb(1, 0, 0, 0) -- alpha=1 for hit testing)
       - Sets cursor via `InputSystemCursor.Create(InputSystemCursorShape.SizeWestEast)` for vertical or `SizeNorthSouth` for horizontal
       - Attaches PointerPressed/PointerMoved/PointerReleased handlers:
         - PointerPressed: CapturePointer, record start position, set e.Handled=true
         - PointerMoved: If left button pressed, compute new ratio from pointer position relative to the split's containing rect, call `_viewModel.SetSplitRatio(nodeId, newRatio)`, set e.Handled=true
         - PointerReleased: ReleasePointerCapture, set e.Handled=true
       - Also handle PointerCaptureLost to reset any drag state
       - Positions handle using Margin (same absolute positioning pattern as pane containers)
       - Adds handle to RootContainer.Children and _resizeHandles list

    3. Call `CreateResizeHandles()` at the END of `RenderLayoutAsync()` (after all pane containers are added, so handles render on top / higher z-index)

    4. To find split boundaries: create a private helper `CollectSplitBoundaries(LayoutNode node, double x, double y, double w, double h, List<(SplitNode, double boundaryPos, double crossStart, double crossSize)>)` that walks the tree similarly to ComputeRectsRecursive. For a vertical split at position (x,y,w,h) with ratio r, the boundary is at x + w*r. For horizontal, at y + h*r.

    **PaneCommandBindings.cs** -- Add swap accelerators in the Attach method:
    - Ctrl+Alt+Shift+Left: `viewModel.SwapActivePane(Direction.Left)`
    - Ctrl+Alt+Shift+Right: `viewModel.SwapActivePane(Direction.Right)`
    - Ctrl+Alt+Shift+Up: `viewModel.SwapActivePane(Direction.Up)`
    - Ctrl+Alt+Shift+Down: `viewModel.SwapActivePane(Direction.Down)`
    Use `VirtualKeyModifiers.Control | VirtualKeyModifiers.Menu | VirtualKeyModifiers.Shift`. Each returns `Task.CompletedTask`.

    Dark theme: resize handles are transparent so no theme concern. The cursor is system-provided.

<task type="auto">
  <name>Task 2: Drag-to-rearrange with blue directional preview overlay</name>
  <files>src/Wcmux.App/Views/WorkspaceView.xaml.cs</files>
  <action>
    Add drag-to-rearrange functionality to WorkspaceView.xaml.cs:

    1. Add fields:
       - `private Border? _dragPreview;` -- semi-transparent blue overlay
       - `private bool _isDraggingPane;` -- drag state flag
       - `private string? _dragSourcePaneId;` -- which pane is being dragged
       - `private Windows.Foundation.Point _dragStartPoint;` -- initial press point for 5px threshold
       - `private const double DragThreshold = 5.0;` -- pixels before drag initiates

    2. Create `InitDragPreview()` called once during initialization:
       - Creates a Border with Background = SolidColorBrush(Color.FromArgb(80, 50, 130, 240)) (semi-transparent blue)
       - Sets IsHitTestVisible = false
       - Sets Visibility = Collapsed
       - Adds to RootContainer.Children (will be on top)

    3. In the existing pane title bar creation (inside the method that creates the 24px pane title bar Grid), attach pointer events for drag initiation:
       - Find where the title bar Grid is created for terminal panes (the 24px bar with process name and buttons)
       - Add PointerPressed handler on the title bar Grid:
         - Record _dragStartPoint, _dragSourcePaneId = paneId
         - CapturePointer on the title bar element
         - Do NOT set _isDraggingPane yet (wait for threshold)
         - Set e.Handled = true
       - Add PointerMoved handler on the title bar Grid:
         - If not yet dragging: check if distance from _dragStartPoint > DragThreshold. If so, set _isDraggingPane = true
         - If dragging: call HitTestDropZone to determine target pane and direction, then update _dragPreview position/size/visibility
         - Set e.Handled = true
       - Add PointerReleased handler on the title bar Grid:
         - If _isDraggingPane: get final drop zone, call `_viewModel.MovePane(_dragSourcePaneId, targetPaneId, dropSide)`
         - Reset all drag state, hide _dragPreview
         - ReleasePointerCapture
         - Set e.Handled = true
       - Add PointerCaptureLost handler to reset drag state

    4. Create `HitTestDropZone(Windows.Foundation.Point pos)` returning `(string? targetPaneId, Direction dropSide)`:
       - Iterate over `_viewModel.LayoutStore.PaneRects`
       - Find which pane rect contains the pointer position
       - Skip if target == source
       - Determine direction using quadrant logic:
         - relX < 0.25 → Left
         - relX > 0.75 → Right
         - relY < 0.5 → Up (when not in left/right zone)
         - else → Down
       - Return target paneId and direction

    5. Create `UpdateDragPreview(string targetPaneId, Direction dropSide)`:
       - Get target PaneRect from LayoutStore.PaneRects
       - Position _dragPreview to cover the half of the target pane indicated by dropSide:
         - Left: left half of target rect
         - Right: right half of target rect
         - Up: top half of target rect
         - Down: bottom half of target rect
       - Set Margin for absolute positioning, Width/Height for size
       - Set Visibility = Visible

    6. During active drag-to-rearrange, hide resize handles to avoid visual clutter (per research recommendation). Show them again on drop/cancel.

    Important: The _dragPreview Border must be added AFTER all pane containers and resize handles in RootContainer.Children so it renders on top. Re-add it at the end of RenderLayoutAsync if it exists.
