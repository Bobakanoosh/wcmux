  <behavior>
    - RaiseBell on unfocused pane sets HasAttention(paneId) to true
    - RaiseBell on focused pane (paneId == activePaneId) is suppressed, HasAttention stays false
    - RaiseBell within 5-second cooldown of same pane is suppressed
    - RaiseBell after cooldown expires raises attention again
    - ClearAttention(paneId) removes attention state
    - HasAttention returns false for unknown paneIds
    - TabHasAttention returns true if any pane in the set has attention
    - TabHasAttention returns false only when ALL panes cleared
    - RemovePane cleans up cooldown and attention state for closed panes
    - AttentionChanged event fires on raise and clear
    - TerminalSurfaceBridge: output containing \u0007 has bell stripped and BellDetected fires
    - TerminalSurfaceBridge: output without \u0007 passes through unchanged, no BellDetected
    - TerminalSurfaceBridge: multiple bells in one batch result in single BellDetected invocation
  </behavior>
  <action>
    1. Add `SessionBellEvent` record to SessionEvent.cs:
       `public sealed record SessionBellEvent(string SessionId) : SessionEvent(SessionId);`

    2. Create `AttentionStore` in src/Wcmux.Core/Runtime/AttentionStore.cs following the immutable-state-with-events pattern from LayoutStore/TabStore:
       - `Dictionary<string, AttentionState> _paneAttention` for current state
       - `Dictionary<string, DateTimeOffset> _lastBellTime` for cooldown tracking
       - `static readonly TimeSpan Cooldown = TimeSpan.FromSeconds(5)`
       - `event Action<string>? AttentionChanged` fires with paneId on state change
       - `bool HasAttention(string paneId)` reads current state
       - `bool TabHasAttention(IEnumerable<string> paneIds)` checks if any pane has attention
       - `void RaiseBell(string paneId, string activePaneId, DateTimeOffset now)`:
         - Suppress if paneId == activePaneId (focused pane)
         - Suppress if within cooldown window
         - Otherwise set attention state and update lastBellTime
       - `void ClearAttention(string paneId)` removes state, fires event
       - `void RemovePane(string paneId)` cleans up all state for closed panes
       - Accept `DateTimeOffset now` parameter (not DateTime.Now) to make tests deterministic

    3. Add `BellDetected` event to TerminalSurfaceBridge:
       - `public event Action? BellDetected;`
       - In `RunOutputBatchLoop`, after building the batch string from StringBuilder:
         - Check `batch.Contains('\u0007')`
         - If found: `batch = batch.Replace("\u0007", "")` then `BellDetected?.Invoke()`
         - Continue with stripped batch for surface delivery
       - This goes BEFORE the existing `_writeToSurface(batch)` call

    4. Write tests FIRST (TDD):
       - AttentionStoreTests.cs: Test all behaviors listed above. Use `[Trait("Category", "Attention")]`.
         Use `DateTimeOffset.UtcNow` and `+ TimeSpan.FromSeconds(6)` for cooldown tests.
       - BellDetectionTests.cs: Create a TerminalSurfaceBridge with a mock write delegate that captures output.
         Enqueue output containing \u0007, wait for batch, assert BellDetected fired and output has bell stripped.
         Use `[Trait("Category", "Attention")]`.
  </action>

    2. Subscribe to `SessionManager.SessionEventReceived` -- when a `SessionBellEvent` arrives, call `_attentionStore.RaiseBell(paneId, activePaneId, DateTimeOffset.UtcNow)`.
       - To map sessionId -> paneId: the WorkspaceViewModel has LayoutStore which maps pane to session via GetSessionId. Iterate AllPaneIds to find the pane whose session matches. Cache this or add a reverse lookup.
    3. Subscribe to TerminalSurfaceBridge.BellDetected for each bridge -- fire SessionBellEvent through SessionManager so it flows through the existing event bus.
       - The bridge is created in WorkspaceViewModel. When a new bridge is created, subscribe its BellDetected event. The handler should fire `SessionManager.SessionEventReceived` with a new `SessionBellEvent(sessionId)`. This may require adding a method to SessionManager to raise arbitrary events, OR have the MainWindow/TabViewModel wire the bridge.BellDetected -> attentionStore.RaiseBell directly, using the bridge's associated sessionId.
       - Simpler approach: In WorkspaceViewModel (which creates bridges), subscribe to each bridge's BellDetected. WorkspaceViewModel knows the sessionId. Fire a new event `BellRaised(string sessionId)` that MainWindow can subscribe to. MainWindow then calls `_attentionStore.RaiseBell(...)`.
    4. Pass `AttentionStore` reference to WorkspaceView and TabBarView (via constructor or attach method)
    5. Subscribe to LayoutStore.ActivePaneChanged -- call `_attentionStore.ClearAttention(paneId)` when a pane is focused
    6. On pane close (LayoutStore change where pane removed), call `_attentionStore.RemovePane(paneId)`

    **WorkspaceView.xaml.cs:**
    1. Accept AttentionStore reference (add parameter to AttachAsync or set as property)
    2. Subscribe to `AttentionStore.AttentionChanged` -- marshal to UI thread via `DispatcherQueue.TryEnqueue`
    3. Modify `UpdateActivePaneHighlight` (or create new `UpdatePaneVisualState`):
       - **Active pane:** Opacity = 1.0, no colored border (use transparent or very subtle border)
       - **Inactive pane without attention:** Opacity = 0.5, use `_inactiveBorderBrush` (existing gray)
       - **Inactive pane with attention:** Opacity = 0.5, blinking blue border
    4. Implement blink animation for attention border:
       - Use `DispatcherTimer` at 500ms interval, toggle border brush between attention blue (`Color.FromArgb(255, 50, 130, 240)`) and transparent
       - Count 4 toggles (2 full blinks ~ 3-5 visible flashes), then stop timer and leave border steady blue
       - Track active blink timers per pane in a `Dictionary<string, DispatcherTimer>`
       - Stop and remove timer when attention clears or pane closes
    5. Apply opacity to the pane container Grid (the one holding Border + TerminalPaneView), not just the border

    **TabBarView.xaml.cs:**
    1. Accept AttentionStore reference (via Attach method alongside TabViewModel)
    2. In `RenderTabs`, for each tab:
       - Get all pane IDs from `tabState.Layout.AllPaneIds`
       - Check `_attentionStore.TabHasAttention(paneIds)`
       - If attention: set tab TextBlock foreground to attention blue, start blink animation
    3. Tab text blink: same DispatcherTimer pattern as pane borders
       - Toggle foreground between attention blue and default color
       - 4 toggles then steady attention color
       - Clear when RenderTabs runs and TabHasAttention returns false
    4. Track blink timers per tab in dictionary, clean up on re-render

    **Threading:** All AttentionStore.AttentionChanged callbacks arrive on background threads. Every UI update must go through `DispatcherQueue.TryEnqueue`. The existing codebase uses this pattern for LayoutChanged and ActivePaneChanged.
