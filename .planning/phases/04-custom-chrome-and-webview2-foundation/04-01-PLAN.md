**MainWindow.xaml changes:**
1. Add a new Row 0 to the root Grid for the title bar (Height="48"). Push TabBarView to Row 1 and TabContentArea to Row 2.
2. The title bar row is a Grid named `AppTitleBar` with Background="#1e1e1e" and three columns:
   - Column 0 (Auto): A FontIcon glyph (Segoe MDL2 Assets, glyph E756 or similar terminal icon) at 16x16, Margin="16,0,0,0", Foreground="#cccccc", VerticalAlignment="Center". Use a FontIcon rather than an Image asset (placeholder per Claude's discretion).
   - Column 1 (Auto): TextBlock "wcmux", Foreground="#cccccc", FontSize="12", FontWeight="Normal", VerticalAlignment="Center", Margin="12,0,0,0", IsHitTestVisible="False".
   - Column 2 (*): Empty -- drag region filler. No content needed.

**MainWindow.xaml.cs changes:**
1. In the constructor, AFTER `InitializeComponent()`, add:
   - `ExtendsContentIntoTitleBar = true;` (MUST be in code, not XAML)
   - `AppWindow.TitleBar.PreferredHeightOption = Microsoft.UI.Windowing.TitleBarHeightOption.Tall;` (for ~48px height)
   - Caption button dark-theme colors via AppWindow.TitleBar:
     - ButtonBackgroundColor = Colors.Transparent
     - ButtonForegroundColor = Color.FromArgb(255, 204, 204, 204)
     - ButtonHoverBackgroundColor = Color.FromArgb(255, 45, 45, 45)
     - ButtonHoverForegroundColor = Colors.White
     - ButtonPressedBackgroundColor = Color.FromArgb(255, 55, 55, 55)
     - ButtonPressedForegroundColor = Colors.White
     - ButtonInactiveBackgroundColor = Colors.Transparent
     - ButtonInactiveForegroundColor = Color.FromArgb(255, 100, 100, 100)
   - Subscribe to `AppTitleBar.Loaded` and `AppTitleBar.SizeChanged` events for region recalculation.

2. Add a `SetRegionsForCustomTitleBar()` method called from both Loaded and SizeChanged handlers:
   - Get `scaleAdjustment` from `AppTitleBar.XamlRoot.RasterizationScale`.
   - Currently no interactive elements in the title bar (the icon and text are not interactive), so no passthrough regions are needed yet. But add the method scaffold so future phases (Phase 5+) can register passthrough regions for buttons added to the title bar.
   - Get `InputNonClientPointerSource.GetForWindowId(AppWindow.Id)` and store it. Even though no passthrough rects are needed now, calling `SetRegionRects(NonClientRegionKind.Passthrough, Array.Empty<Windows.Graphics.RectInt32>())` establishes the pattern.

3. Add required `using` statements:
   - `using Microsoft.UI;`
   - `using Microsoft.UI.Input;`
   - `using Microsoft.UI.Windowing;`
   - `using Windows.Graphics;` (for RectInt32)

**Anti-patterns to avoid (from research):**
- Do NOT use `Window.SetTitleBar()` -- locked decision to use InputNonClientPointerSource.
- Do NOT set ExtendsContentIntoTitleBar in XAML -- must be in code-behind.
- Do NOT set PreferredHeightOption before ExtendsContentIntoTitleBar -- throws exception.
- Do NOT calculate regions before Loaded event -- dimensions not available yet.
