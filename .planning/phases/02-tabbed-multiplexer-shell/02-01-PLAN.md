<feature>
  <name>TabStore: Core tab collection state management</name>
  <files>src/Wcmux.Core/Layout/TabStore.cs, tests/Wcmux.Tests/Layout/TabStoreTests.cs</files>
  <behavior>
    - CreateTab: Creates a new tab with a fresh LayoutStore, sets it as active, adds to ordered tab list, fires TabsChanged and ActiveTabChanged events
    - CreateTab: Tab label defaults to provided label string, IsCustomLabel is false
    - SwitchTab: Changes ActiveTabId, fires ActiveTabChanged, does NOT modify any tab's LayoutStore
    - SwitchTab: Calling with current active tab ID is a no-op (no event fired)
    - SwitchTab: Calling with unknown tab ID throws or is no-op
    - CloseTab: Removes tab from collection and order list, fires TabsChanged
    - CloseTab: If closed tab was active, switches to adjacent tab (prefer right neighbor, then left)
    - CloseTab: Returns the closed tab's TabState so caller can dispose resources
    - CloseTab: When closing the last tab, fires LastTabClosed event and returns the tab state
    - RenameTab: Updates label and sets IsCustomLabel to true, fires TabsChanged
    - TabOrder: Maintains insertion order, reflects in IReadOnlyList
    - GetTab: Returns TabState by ID, null for unknown ID
    - ActiveLayout: Convenience property returning the active tab's LayoutStore
    - TabCount: Returns number of tabs
  </behavior>
  <implementation>
    Create TabStore as a sealed class in Wcmux.Core.Layout namespace. Use a Dictionary for tab lookup and List for ordering.

    TabState record: TabId (string), Layout (LayoutStore), Label (string), IsCustomLabel (bool).

    Events: TabsChanged (Action), ActiveTabChanged (Action with string tabId), LastTabClosed (Action).

    Follow the established LayoutStore pattern: mutable internal state, events for observation, no UI dependencies.

    The CloseTab method should handle the "last tab" case by firing LastTabClosed instead of trying to switch to a nonexistent tab.
  </implementation>
</feature>

    - TruncateCwdFromLeft: short paths (under maxLength) returned as-is after home replacement
    - TruncateCwdFromLeft: long paths truncated from left with ".../" prefix showing last segments
    - TruncateCwdFromLeft: single-segment path never truncated
    - TruncateCwdFromLeft: UNC paths handled without crash
    - FormatTabLabel: home directory returns "~", otherwise returns last 2 segments of path
  </behavior>
