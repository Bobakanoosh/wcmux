3. Create `ForegroundProcessDetector.cs` in `src/Wcmux.Core/Runtime/` using the ToolHelp32 P/Invoke pattern from the research document. Key points:
   - Use `CreateToolhelp32Snapshot` with `TH32CS_SNAPPROCESS = 0x00000002`
   - Define `PROCESSENTRY32` struct with `CharSet.Auto` and `SizeConst = 260` for `szExeFile`
   - Build `Dictionary<uint, List<(uint pid, string name)>>` parent-to-children map from single snapshot
   - Walk from shellPid to deepest child, return `Path.GetFileNameWithoutExtension(name)`
   - CRITICAL: Always `CloseHandle` the snapshot in a finally block to prevent handle leaks
   - Read process name from `PROCESSENTRY32.szExeFile` (NOT from Process.GetProcessById) to avoid race conditions
   - Return null for invalid/zero PID or empty process tree
